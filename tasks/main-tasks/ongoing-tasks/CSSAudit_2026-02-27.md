| ID | Status | Priority | Notes |
|---|---|---|---|
| RESP-01 | ❌ FAIL | Medium | `width: 300px` on `.stwid--list` — fixed px width on a panel (line 144) |
| RESP-02 | ❌ FAIL | Medium | `width: 300px` on `.stwid--list` lacks accompanying `max-width: 100%` (line 144) |
| RESP-03 | ❌ FAIL | High | `width: 300px` on `.stwid--list` fills a major layout region (line 144) |

### RESP-01 / RESP-02 / RESP-03 — FAIL (High, Medium, Medium)

All three rules are violated by the same declaration:

**Violation** (line 144):
```css
.stwid--list {
  flex: 0 0 auto;
  display: flex;
  flex-direction: column;
  overflow: hidden;
  width: 300px;
  min-width: 0;
  min-height: 0;
}
```

The list panel is a major side-by-side layout region. Using a fixed `width: 300px` means on narrow-but-still-desktop viewports (1001–300px) the panel can overflow. `max-width: 100%` is also absent.

**Corrected:**
```css
.stwid--list {
  flex: 0 0 auto;
  display: flex;
  flex-direction: column;
  overflow: hidden;
  width: 20vw;          /* fluid unit; adjust to taste */
  min-width: 220px;     /* prevent panel from becoming too narrow */
  max-width: 100%;      /* RESP-02 requirement */
  min-height: 0;
}
```
Or keep `width: 300px` and add `max-width: 100%` as a minimal fix for RESP-02, while accepting that RESP-01 and RESP-03 remain partially unresolved.

## ANIM — Animation

| ID | Status | Priority | Notes |
|---|---|---|---|
| ANIM-01 | ❌ FAIL | High | `margin-bottom` (line 1452) and `max-height` (line 1478) transitions animate layout-tier properties |
| ANIM-02 | ❌ FAIL | Medium | 7 bare `transition: 200ms` declarations — no property named (lines 659, 752, 795, 830, 1708, 1860, 1910) |
| ANIM-03 | — N/A | — | No `will-change` usage in the file |

### ANIM-01 — FAIL (High)

**Violation — violation 1** (line 1452):
```css
.stwid--RowTitle {
  transition: margin-bottom 250ms ease;
}
```
`margin-bottom` causes layout reflow on every frame.

**Corrected:** Replace margin animation with `gap` on the parent flex container (already available), or use `padding` only if the container has `overflow: hidden`. Better yet, animate `opacity` and `max-height` only — but see ANIM-01 note for `max-height` below. Alternatively use transform on a pseudo-element spacer.
```css
/* Remove the transition from RowTitle entirely, since the visible change
   is the chevron rotation (ANIM-01-safe) and the content wrap collapsing */
.stwid--RowTitle {
  /* no transition needed — margin-bottom is toggled by class, not smoothed */
}
```

**Violation — violation 2** (line 1478):
```css
.stwid--rowContentWrap {
  transition: max-height 250ms ease;
}
```
`max-height` animation causes layout reflow. The element lacks an explicit `max-height` in its default state, meaning the transition cannot run smoothly.

**Corrected:** Replace with a `clip-path` or `transform: scaleY()` + `transform-origin: top` approach, or use the CSS `@keyframes` with `display: contents` toggle and `@starting-style`:
```css
.stwid--rowContentWrap {
  /* Remove max-height transition; use opacity + visibility for a safe fade: */
  transition: opacity 200ms ease;
}
.stwid--collapsed .stwid--rowContentWrap {
  opacity: 0;
  pointer-events: none;
}
```

### ANIM-02 — FAIL (Medium)

Bare `transition: 200ms` declarations apply to **all** properties by default, making them equivalent to `transition: all 200ms`. This is a violation of ANIM-02 and compounds ANIM-01 by potentially catching layout properties.

**7 violations** (lines 659, 752, 795, 830, 1708, 1860, 1910):
```css
transition: 200ms;
```

**Corrected** — for each, name the specific property being transitioned. For example:

| Line | Context | Corrected |
|---|---|---|
| 659 | Folder/book action icon opacity | `transition: opacity 200ms ease;` |
| 752 | `.stwid--entry` | `transition: background-color 200ms ease;` |
| 795 | `.stwid--entry .stwid--selector` | `transition: opacity 200ms ease;` |
| 830 | `.stwid--entry .stwid--status` | `transition: opacity 200ms ease;` |
| 1708 | `stwid--orderTable tr` | `transition: background-color 200ms ease;` |
| 1860 | `.stwid--sortableHandle` | `transition: opacity 200ms ease;` |
| 1910 | `.stwid--orderSelect` | `transition: opacity 200ms ease;` |

### ACC-02 — FAIL (High)

The file has no `@media (prefers-reduced-motion: reduce)` block anywhere. Users with motion sensitivity receive all transitions regardless of their system preference.

The following non-opacity transitions require a reduced-motion override:

| Line | Transition |
|---|---|
| 358 | `opacity 0.2s ease, color 0.2s ease, border-color 0.2s ease` |
| 449 | `color 150ms ease, background-color 150ms ease` |
| 588 | `background-color 150ms ease, box-shadow 150ms ease` |
| 659, 752, 795, 830, 1708, 1860, 1910 | `200ms` (bare — see ANIM-02) |
| 1177 | `background-color 200ms, color 200ms, opacity 200ms` |
| 1452 | `margin-bottom 250ms ease` |
| 1465 | `transform 250ms ease` |
| 1478 | `max-height 250ms ease` |

**Corrected:** Add a single `prefers-reduced-motion` block after all animated sections, or inline after each animated section:
```css
@media (prefers-reduced-motion: reduce) {
  /* Disable all non-opacity transitions */
  *,
  *::before,
  *::after {
    transition-duration: 0.01ms !important; /* override: blanket motion kill for reduced-motion preference */
    animation-duration: 0.01ms !important;  /* override: blanket motion kill for reduced-motion preference */
  }
}
```
Or, for surgical precision, override each named transition individually.
